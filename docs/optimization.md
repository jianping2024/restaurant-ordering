# Cursor 项目整理阶段指挥文档

## 总原则

每次只执行一个阶段。  
每个阶段完成后必须停止，等待我确认。  
没有确认，不允许进入下一阶段。

Cursor 必须遵守：

1. 先读文档和代码，再给方案
2. 不允许直接大规模重构
3. 不允许顺手改无关文件
4. 不允许新增需求外功能
5. 不允许用临时补丁绕过结构问题
6. 每次执行后必须输出修改文件、影响范围、测试结果和风险点

---

# 阶段 0：项目现状审计

## 目标

先看清楚当前项目是什么样，不改任何代码。

重点搞清楚：

1. 当前有哪些功能模块
2. 当前代码目录怎么组织
3. 哪些模块最混乱
4. 哪些地方重复造轮子
5. 哪些需求和业务规则缺文档
6. 哪些模块重构风险最高

## 给 Cursor 的指令

```txt
请执行阶段 0：项目现状审计。

只读项目，不要修改任何文件，不要新增文件，不要格式化代码。

请完成以下任务：
1. 梳理当前系统已有功能模块
2. 分析当前项目目录结构
3. 找出代码结构不合理的位置
4. 找出重复组件、重复逻辑、临时补丁
5. 找出高风险模块
6. 找出缺少需求文档或业务规则文档的位置

输出一份《项目现状审计报告》，必须包含：
1. 当前功能模块清单
2. 当前目录结构分析
3. 高风险模块列表
4. 重复代码和重复组件初步判断
5. 文档缺失清单
6. 代码结构主要问题
7. 下一阶段建议

完成后停止，等待我确认。

```

## 验收标准

Cursor 没有改任何文件，并且输出了完整审计报告。

```txt
根据你的阶段 0 审计报告，现在执行阶段 1：建立文档骨架和 AI 规则。
```

不要一次性说：

```txt

```

这会让 Cursor 失控。

正确做法是：

```txt
只执行阶段 0。

```
