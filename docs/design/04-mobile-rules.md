# 移动端规则

> **状态**：阶段 4 已填充（2026-06-30）  
> **读者**：设计、前端开发、AI 代理

## 用途

餐馆现场 **手机** 操作的专项约束。与 [`01-design-principles.md`](./01-design-principles.md) 移动端优先原则配套执行。

---

## 1. 手机端不要塞复杂表格

### 禁止

- 服务员看板用多列表格扫桌态
- 按菜分单用「行=菜、列=客人」矩阵
- 结账详情在首屏堆完整订单宽表

### 替代

| 场景 | 移动布局 |
|------|----------|
| 桌位状态 | 单列卡片网格，颜色 + 桌号 + 副标题 |
| 按菜分单 | `ByItemDishAllocator` 每菜一张卡，纵向列表 |
| 结账菜品 | 默认折叠「本桌菜品」，点开才展开 |
| 订单历史 | 订单卡片 + chips；筛选器堆叠 |
| 经营分析 | 允许表横向滚动，但 Top 列表优先简列 |

### 验收

在 **375px 宽** 视口下无需横向滚动即可完成：点餐提交、呼叫结账、确认单笔收款。

## 2. 高频操作路径要短

### 目标路径（点击次数参考）

| 角色 | 任务 | 路径 |
|------|------|------|
| 顾客 | 加菜并提交 | 扫码 → 点菜品 → 购物车 → 提交（≤4 步） |
| 顾客 | 呼叫结账 | 账单页 → 选分单 → 确认（≤3 步） |
| 服务员 | 开台 | 看板 → 桌台 → 确认开台（≤3 步） |
| 收银员 | 收一桌钱 | 结账 → 选桌 → 点收款（≤3 步） |

### 规则

- 少全页跳转：购物车用 **抽屉**；结账手机用 **列表↔详情** 而非新路由
- 服务员点餐提交后 **立即自动回桌台**（无 success toast；见 `menu-order-submit-outcome.ts`）
- Dashboard 手机：**汉堡菜单** 一次展开导航，避免每页重复顶栏按钮

### 避免

- 仅为「好看」增加中间落地页
- 把打印、关台与收款放在同一拥挤底栏 — 收款区优先，次要操作放底部次要行

---

## 3. 分单人数多时不要全部平铺

### 均摊（even）

- 消费者姓名列表纵向排列；姓名输入用 `ConsumerNameCombobox`
- 人数 >6 时考虑 **折叠未分配** 或分页 — 当前实现纵向列表；新增勿改横向挤一排头像

### 按菜（by_item）

- **一道菜一张卡**；每人一行分配，不把所有客人列横滑
- 顶部 **进度条** `complete/total` 一眼看是否分完
- 未完成：进度条 `red-500`；完成：`emerald-500`

### 结账待收款

- 每人一行：姓名 + **本次金额** + 收款按钮
- 不采用「九宫格客人按钮」以免金额看不清

---

## 4. 点餐和结账按钮必须明显

### 点餐

| 元素 | 规范 |
|------|------|
| 购物车入口 | 底部贴底固定条（栏内 safe area），显示件数 + 合计 |
| 提交订单 | 抽屉内 **全宽 gold** `提交订单` / `Place order` |
| 禁用态 | `disabled:opacity-50` + 门禁文案（未开台等） |

### 结账

| 元素 | 规范 |
|------|------|
| 呼叫结账 | `BillPage` 底栏 gold 主按钮 |
| 确认收款 | `mesa-badge-success`，文案含 **€金额** |
| 待收金额 | 列表卡片 **最大数字 = 待收**（非消费总额） |

### 对比度

- 主按钮不得用 `ghost` 或纯文字
- 与背景对比：gold 主钮在 `brand-card` 上；收款绿钮在 gold 浅底区块上

---

## 5. 状态反馈必须清楚

### 加载

- 按钮 `loading` 旋转图标；列表用骨架或「加载中」文案
- 结账 Realtime 刷新时勿整页闪白 — 局部更新 requests

### 成功

- 加菜：`showToast` 成功
- 收款：Toast + 列表/详情移除该人待收行
- 开台：详情刷新或乐观更新后对齐服务端

### 失败

- Toast 错误 + 可理解 i18n（`session_billing`、`rate_limited` 等映射）
- 退菜/原因类：对话框 `externalError` 字段内联

### 禁用原因

- 恢复点单、折扣锁定、菜单门禁 — **可见说明文字**，不只 `disabled` 灰掉
- 示例：`guestOrderingActionHint` 区分 `billing` vs 未开台

### 声音（结账）

- 新结账请求可选提示音（`playCheckoutRequestChime`）；设置可关

## 触控与 Safe Area

| 项 | 约定 |
|----|------|
| 最小点击目标 | 约 44×44px（`py-2 px-3` 及以上） |
| 底部固定栏 | 考虑 `pb-safe` / 留底边距，避免 Home 条遮挡 |
| Modal / Drawer | `max-h-[calc(100dvh-*)]` 使用 dynamic viewport |
| 顶栏 | Dashboard 手机 `h-16` 固定；内容 `pt-20` |

## 主要移动端页面检查表

| 页面 | 必查项 |
|------|--------|
| `MenuPage` | 底栏购物车、分类 Tab 可横滑、语言切换可达 |
| `BillPage` | 分单进度、主按钮固定、锁定态有说明 |
| `WaiterDisplay` | 筛选 chip、桌卡可点区域够大、待结账色明显 |
| `WaiterTableDetail` | 自助餐步进器、订单列表不截断主按钮 |
| `CheckoutRequestsManager` | 返回列表、收款钮、金额字号 |
| `KitchenDisplay` | 大行可点、void 确认不挡下一单 |
