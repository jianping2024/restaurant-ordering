# 恢复点单与分单续结（需求口径）

> 结账收款 UI：`CheckoutRequestsManager`、顾客账单 `BillPage`  
> 数据：`session_collected_payments`、`bill_splits`（`split_mode`、`persons`、`result`）  
> 相关 RPC：`resume_table_session_ordering`、`confirm_bill_split_payment`  
> 关台语义（强制结束餐次）：[`table-session-close.zh.md`](./table-session-close.zh.md)

## 1. 场景

一桌客人已**呼叫结账**，并选择了**按菜分单**（或其它分单模式）。服务员在后台**已确认部分客人收款**（写入「已收款项」），但：

- 仍有客人未付清，或
- 客人/服务员需要**继续加菜**，

于是执行 **恢复点单**：餐次从 `billing` 回到 `open`，允许继续下单。

恢复点单的业务含义是 **「续餐续结」**，不是作废本餐次已发生的结账进度。

## 2. 核心原则

| 原则 | 说明 |
|------|------|
| **已收款项不可动** | `session_collected_payments` 为本餐次收款台账；恢复点单**不得**冲销、覆盖或丢失已有记录。再次结账时须完整展示历史收款与合计。 |
| **已确定的分单须记住** | 已呼叫结账时写入的 `bill_splits` 分单方案（模式、客人、按菜归属、每人应付）在部分收款后**视为已生效**；恢复点单后再次结账须**沿用**，而非从零重选。 |
| **分单模式锁定** | 本餐次一旦存在**按菜分单且已有部分收款**，则**不允许**再切换均摊 / 按菜 / 自定义，也不允许通过改人数、改客人名单等方式推翻原方案。 |
| **已收款对应菜品冻结** | 已确认收款的客人，其名下已结清部分的菜品分配**只读**：不可改归属、不可改份额、不可划给他人。 |
| **未收款部分方案仍有效** | 尚未确认收款的客人，原先分给他的菜与应付金额**继续有效**；再次结账时仍按原方案待收（加菜或折扣变更时按规则算差额，见 §4）。 |
| **整桌已收不可恢复** | 若整桌（单条总计分账）已收款，或业务上视为整桌已结清，**禁止**恢复点单（与现有 `whole_table_paid` 拦截一致）。 |

## 3. 服务员侧结账详情（`CheckoutRequestsManager`）

与 [`research.md`](../research.md) §9 一致，并补充续结语义：

- **已收款项**：展示本餐次全部历史确认收款（含恢复点单之前发生的）。
- **分单结果**：仅展示**尚未确认收款**的客人；已收款者不在此重复出现。
- **恢复点单**后再次进入结账详情：已收款项与待收名单须与恢复前一致（仅因新加菜而增加待结金额或新增待分配行，见 §4）。

## 4. 按菜分单（`split_mode = by_item`）续结规则

### 4.1 恢复点单前已存在的菜

- **已收款客人**：其 `persons[].item_shares`（及对应 `result` 行 `paid: true`）所覆盖的订单行 **锁定**。
- **未收款客人**：原有按菜归属 **保留**；再次结账时仍出现在「分单结果」待收列表中。

### 4.2 恢复点单后新加的菜

- 新订单行纳入**同一餐次**、**同一按菜分单框架**。
- 仅对 **新增行** 允许分配消费者（分给哪位客人）；**不得**改动已锁定行的归属。
- 新菜金额并入对应客人的应付；服务员确认收款时，「建议本次收」须扣除该客人历史已收（`session_collected_payments` + `suggestedCollectionAmount` 语义）。

### 4.3 顾客账单页（`BillPage`）

- 若本餐次已存在「按菜分单 + 部分收款」：
  - **隐藏或禁用**分单模式切换（均摊 / 按菜 / 自定义）。
  - 按菜分配区：已收款客人已锁定的行 **只读**；未收款客人原有分配只读；**仅新菜行可编辑**。
- 若仅为首次结账、无任何收款：行为与现网一致（可选模式、可编辑分配）。

### 4.4 未决产品边界（实现前须拍板）

- **未收款客人的「旧菜」**：默认 **不重分**，仅新菜可编辑（§4.1–4.2）。若未来允许「未收部分整体重分、仅已收部分锁定」，须单独开需求并改本文档。

## 5. 与其它分单模式

| 模式 | 部分收款后恢复点单 |
|------|-------------------|
| **整桌总计**（单行 / 无分人） | 已有收款或台账非空 → **禁止**恢复点单（现状）。 |
| **均摊 / 自定义** | 已有部分收款后，分单模式与人数/金额方案 **锁定**；仅允许在新增消费后调整**未收款客人**的待收金额（折扣、加菜引起），**不可**改模式或推翻已 `paid` 行。按菜专属的「逐行锁定」不适用，但 `result[].paid` 行仍不可再收、不可从待收列表删改含义。 |

## 6. 允许与禁止（一览）

**允许**

- 继续点新菜、加单。
- 对**新增菜**（按菜模式）做分配。
- 对**未收款**客人继续「确认收款」。
- 查看已收款项与待收名单。

**禁止**

- 切换分单模式（在已部分收款后）。
- 修改已收款客人已锁定的菜品归属或份额。
- 丢失或篡改 `session_collected_payments`。
- 将部分收款后的恢复点单当作「取消分单、重新结账」。
